Не в каждом файле <processor> находится внутри <device>. Иногда он объявлен глобально внутри <family>

Некоторые устройства, такие как STM32 содержат внутри <family> ветку <subFamily>, в то время как Nordic такой ветки не содержит

В CoIDE принято такое разделение:
-- Manufacturer -> Family -> Serie -> Chip

Если провести аналогию с pdsc, то :
Manufacturer = <family Dvendor="STMicroelectronics:13"> (13 - это Vendor ID)
Family = <processor Dcore="Cortex-M4">
Serie = <subFamily DsubFamily="STM32F401"> или <family Dfamily="nRF51 Series"> (если нет subFamily)
Chip = <device Dname="nRF51801_xxAB">

То, что внесено в базу как Family, Serie и Chip средой автоматически генерируется в качестве Define, т.е. -DSTM32F401;

ID Manufacturer в базе CoIDE не соответствуют Vendor ID в системе Keil

Family в структуре CoIDE должны быть переименованы с "Cortex-M0" на "Cortex M0" (иногда автоматически вставляется дефайн -DM0;)

Алгоритмы программирование могут быть заданы как для семейства, так и для отдельного устройства. Например, Nordic DFP не содержит алгоритмов внутри device

значение default=1 может быть отмечено для нескольких алгоритмов (если они шьют разные регионы). Надо отслеживать

Вот так можно в Windows создавать ссылки на файлы, чтобы не размножать их копии в качестве компонентов CoIDE:

# Переходим в целевую директорию
cd C:\Projects\MyApp

# Создаем относительную ссылку на файл
mklink config.json ..\Shared\config.json

# Создаем относительную ссылку на папку
mklink /D node_modules ..\..\Common\node_modules


for %i in (..\..\..\..\..\..\ThirdParty\Delfino_HAL\lib\buffer\*.h) do mklink "%~ni.h" "%i"
